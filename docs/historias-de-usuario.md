## Contexto de la actividad
SwimTrack es una plataforma web orientada a conectar entrenadores de
natacion con usuarios interesados en servicios de entrenamiento personalizado.
Las funcionalidades asociadas a las siguientes historias de usuario ya se
encuentran contempladas en la version base del proyecto. Para la presente
actividad, el equipo documenta dichas historias dentro del repositorio y
evidencia el uso colaborativo de Git y GitHub.
## HU-01 - Registro de usuario
**Como** usuario no registrado, **quiero** crear una cuenta con mi nombre,
correo y contrasena, **para** acceder a los servicios de entrenamiento.
### Criterios de aceptacion
- El formulario solicita nombre, correo electronico y contrasena.
- El sistema valida los datos requeridos.
- El usuario puede continuar al proceso de autenticacion.
## HU-02 - Inicio de sesion
**Como** usuario registrado, **quiero** iniciar sesion con mi correo y
contrasena, **para** acceder a las funcionalidades de SwimTrack.
### Criterios de aceptacion
- El sistema solicita correo electronico y contrasena.
- Las credenciales correctas permiten el acceso.
- La autenticacion se gestiona mediante JWT.
## HU-03 - Busqueda de entrenadores
**Como** usuario cliente, **quiero** visualizar entrenadores disponibles,
**para** identificar una opcion de entrenamiento apropiada.
Plan operativo para grabacion de pantalla - Nicolas (macOS) y Josberth
### Criterios de aceptacion
- El sistema permite consultar entrenadores disponibles.
- Cada entrenador presenta informacion relevante.
- El usuario puede identificar opciones de entrenamiento.
## HU-04 - Perfil de entrenador
**Como** entrenador registrado, **quiero** gestionar mi informacion profesional
y disponibilidad, **para** que los usuarios conozcan mis servicios.
### Criterios de aceptacion
- El entrenador puede registrar o actualizar informacion de su perfil.
- El perfil contiene informacion profesional y disponibilidad.
- Los usuarios pueden consultar la informacion disponible.
## HU-05 - Solicitud de sesion de entrenamiento
**Como** usuario cliente, **quiero** enviar una solicitud de sesion a un
entrenador, **para** iniciar un entrenamiento personalizado.
### Criterios de aceptacion
- El usuario puede seleccionar un entrenador.
- El usuario puede enviar una solicitud de sesion.
- El sistema informa el registro o envio de la solicitud.
